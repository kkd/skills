# インターフェース×人の全体性（Layer U）

## 根本的な前提

**全体性の「場」は人の認知・感情・行動の中にある。**

インターフェースは画面上の要素ではなく、人とプロダクトが「接触する瞬間」として存在する。
この接触の質——スムーズさ・摩擦・驚き・信頼——がプロダクトへの愛着を決定づける。

Layer P が「このプロダクトは自分の生活に奉仕しているか」を問うのに対し、
Layer U は「このインターフェースは今この瞬間、自分に語りかけているか」を問う。

構造保存変容の観点では:
- 変容前に存在していた「使いやすさの感覚」「自然なリズム」を保ちながら変化する
- 利用者が「あれ、どこ行った？」と感じたら、センターが失われた証拠だ
- **「気づいたら変わっていた（でも自然）」が理想の Layer U 変容**

---

## Layer U のセンター構造

```
スーパーセンター（人の認知・目的・文脈）
  │
  └── 画面センター（ページ/ビュー全体の構造）
        │
        ├── 焦点センター（プライマリアクション・視線の起点）
        │
        ├── ナビゲーションセンター（移動・現在地の把握）
        │
        ├── インタラクションセンター（入力→フィードバックの流れ）
        │
        └── 情報センター（読む・理解するコンテンツの構造）
```

| センター | 意味 | 弱さの兆候 |
|---------|------|-----------|
| **画面センター** | ページ全体の視覚的重心と構造 | 「何の画面か」が一目でわからない |
| **焦点センター** | 利用者の注意が自然に向かう核 | 「どこを押せばいいか」迷う |
| **ナビゲーションセンター** | 現在地・移動先の構造 | 「今どこにいるか」わからない |
| **インタラクションセンター** | 操作とフィードバックのリズム | 操作結果が予測できない・遅延を感じる |
| **情報センター** | 読み取りの流れと階層 | 情報が多すぎて何を読めばいいかわからない |

## 15特性の UI/UX 対応表

| 特性 | UI/UX での現れ | 弱さの兆候 |
|------|--------------|-----------|
| 1 スケールの段階性 | 視覚的階層（見出し > 本文 > キャプション）、コンポーネントの粒度差 | すべての要素が同じ視覚的重みを持つ |
| 2 力強いセンター | 画面の主目的が一目でわかるフォーカスポイント | CTAが複数あり、どれが主かわからない |
| 3 境界 | カード・モーダル・セクション区切りの明確さ | コンテンツ領域の境界が曖昧 |
| 4 交互反復 | 入力→確認、問い→答え、アクション→フィードバックのリズム | 一方向の情報提示のみ |
| 5 正の空間 | ホワイトスペースが意味を持つ余白 | 余白がなく詰め込まれた印象 |
| 6 よい形 | ボタン・フォーム要素が操作の「形」を自然に表現 | ボタンがリンクに見える、入力欄がわかりにくい |
| 7 局所的シンメトリー | グリッド整合、アイコンの一貫したスタイル | 要素が整列していない、アイコンの雰囲気がバラバラ |
| 8 深い相互結合 | ホバー・フォーカス・トランジションが操作意図と有機的に結びつく | アニメーションが飾りで操作感覚と無関係 |
| 9 コントラスト | プライマリ/セカンダリ アクションの視覚的差異、文字の可読性 | すべてのアクションが同じ視覚的重さを持つ |
| 10 グラデーション | 初心者→上級者へのプログレッシブディスクロージャー | 全機能を最初から並べる |
| 11 粗っぽさ | プロダクトの個性・語り口（コピーライティング含む） | 無個性で交換可能な印象 |
| 12 共鳴 | デザインシステム、一貫したインタラクションパターン | 画面ごとに操作方法が違う |
| 13 空 | エンプティステート、意図的な余白、「何もしない」状態のデザイン | エラー時・空状態が設計されていない |
| 14 シンプルさと静謐さ | 使い始めて即座に「わかる」感覚、認知負荷の最小化 | 操作手順の説明が必要 |
| 15 不可分性 | インターフェースが利用者の文脈に溶け込む感覚 | 「プロダクトの使い方を覚えなければならない」感 |

## Layer U の弱さのシグナル

| 利用者の言動・行動 | 失われているセンター | 対応する特性 |
|-----------------|-------------------|------------|
| 「どこを押せばいい？」と迷う | 焦点センターが弱い | 特性2（力強いセンター）、特性9（コントラスト） |
| 操作後に何が起きたかわからない | インタラクションセンターが弱い | 特性4（交互反復）、特性8（深い相互結合） |
| 情報を探して画面を往復する | 情報センターの階層が失われている | 特性1（スケール）、特性14（シンプルさ） |
| 「なんか違和感がある」 | 局所的シンメトリーまたは共鳴の欠如 | 特性7、特性12 |
| ツールチップ・ヘルプへの依存 | よい形が実現されていない | 特性6（よい形）、特性15（不可分性） |
| 同じ場所を繰り返しタップ/クリック | インタラクションセンターのフィードバック不足 | 特性4、特性8 |
| 使い方を人に聞く | 共鳴と不可分性の欠如 | 特性12、特性15 |

## センター検出プロトコル（Layer U）

### 定量的分析

```
# アクセス解析・ヒートマップから
- クリック率が低いCTA → 焦点センターが弱い
- 離脱率が高いページ → 画面センターが利用者の目的と接続していない
- 繰り返されるエラー → インタラクションセンターのフィードバック不足
- ヘルプページへの流入 → 情報センターまたはインタラクションセンターが弱い

# ユーザーテスト・インタビューから
- 「迷った場面」の記録 → 弱い焦点センター・ナビゲーションセンター
- 「気持ちよかった場面」の記録 → 強いセンターの手がかり
- 操作を諦めた場面 → 全体性が断ち切られたポイント
```

### 定性的問い

1. **画面を見て0.5秒で「何をする画面か」わかるか？**
   → 画面センターの強さの確認

2. **初めて使う人が説明なしに主要操作を完了できるか？**
   → 焦点センターとインタラクションセンターの健全性

3. **操作のリズムが自然に感じられるか？**
   → インタラクションセンターの特性4・8

4. **この画面の「次の一手」が自然に示されているか？**
   → ナビゲーションセンターと特性10（グラデーション）

5. **空の状態・エラー状態でも全体性が保たれているか？**
   → 特性13（空）の実現

## Layer U × Layer C の張力

コードの変更が Layer U の全体性に影響する主なケース:

| C の変更 | U への影響 |
|---------|-----------|
| コンポーネント分割 | 画面センターの再構成が必要になる |
| 状態管理の変更 | インタラクションセンターのフィードバックタイミングが変わる |
| API レスポンス速度の変化 | 待機状態のデザイン（特性13）が機能しなくなる |
| データ構造の変更 | 情報センターの表示ロジックが変わる |

**C×U の張力を読む問い:**
> 「このコードの変更は、利用者の操作感覚に何を変えるか？」

---

## Layer U × Layer P の張力

インターフェースパターンが利用者の生活文脈から乖離している主なケース:

| U の状態 | P との乖離 |
|---------|-----------|
| 操作ステップが多い | 利用者が「ながら作業」で使う文脈を無視している |
| 情報密度が高い | 利用者が「短時間・移動中」に使う文脈と合っていない |
| 専門用語が並ぶ | 利用者の語彙・メンタルモデルと乖離している |
| デスクトップ優先の UI | 利用者の主要アクセス環境がモバイルである |

**U×P の張力を読む問い:**
> 「この画面を、利用者が実際に使う状況（場所・時間・気持ち）で想像したとき、何が引っかかるか？」

## Layer U の構造保存変容パターン

### 変容の原則

1. **利用者のメンタルモデルを保存する**
   変容後、利用者が「あれ、どこ行った？」と感じたら失敗。
   強いセンターの位置・形・名前は可能な限り維持する。

2. **一度に変えるセンターは1つ**
   複数のセンターを同時に変えると、何が変わったかわからなくなる。
   利用者は全体性の変化を一度に吸収できない。

3. **変容の前後で全体性スナップショットを比較する**
   数値（タスク完了率、エラー率）だけでなく、
   「使ったときの感覚」の変化を人間が感じ取ること。

### 代表的な変容の方向

| 弱い特性 | 変容の方向 |
|---------|-----------|
| 特性2（焦点センターが弱い） | CTAの視覚的重みを上げる。他の要素のコントラストを下げる |
| 特性4（インタラクションリズムが壊れている） | フィードバックアニメーション・ローディング状態を整える |
| 特性14（認知負荷が高い） | 表示情報を削減する。プログレッシブディスクロージャーを導入 |
| 特性15（不可分性が低い） | 利用者の語彙に合わせたコピー変更・ラベル変更 |
